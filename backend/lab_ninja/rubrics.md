# Lab-Ninja Answer-Evaluation Rubrics

This document outlines the evaluation guidelines and expectations for different question sub-types.

## Section A — Core Interview Track

### 1. DSA / Problem Solving
- **Correctness**: The solution must solve the specified problem and handle all edge cases.
- **Complexity**: Time and space complexities must match the optimal expected complexities.
- **Clarity**: The code and explanation must be clean, readable, and structured.

### 2. System Design (Mid-Scale)
- **APIs**: Design clean, logical API endpoints with appropriate request/response bodies.
- **Database Schema**: Propose proper relational/non-relational schemas, indices, and fields.
- **Scalability**: Address caching, replication, and standard mid-scale performance bottlenecks.

### 3. Scenario-Based Technical Questions
- **Root-Cause Analysis**: Address the actual root cause of production bugs or regressions.
- **Engineering Trade-offs**: Make logical choices based on cost, latency, or reliability constraints.

### 4. Large-Scale System Design
- **Scale Constraints**: Estimate bandwidth, storage, memory, and QPS bounds.
- **High Availability**: Incorporate load balancing, auto-scaling, fault tolerance, and consensus components.

### 5. General Interview / Behavioral Questions
- **STAR Method**: Format the response containing Situation, Task, Action, and Result.
- **Clarity & Impact**: Emphasize individual actions, collaboration, lessons learned, and metrics-driven impact.

## Section B — Computer Vision Track

### 1. Classical Computer Vision
- Must demonstrate deep knowledge of camera models, projection matrix, homography, epipolar geometry, filtering, and edge detection.

### 2. Deep Learning for Vision
- Must cover neural network architectures (CNNs, Transformers), loss functions (contrastive loss, IoU loss, etc.), and optimizer selection.

### 3. Training & Evaluation Strategies
- Explain how to handle overfitting, imbalance, data augmentation, hyperparameter tuning, and cross-validation.

### 4. Dataset Quality & Bias
- Address issues of out-of-domain distribution, labelling quality, annotation constraints, and fairness.

### 5. Inference Optimization & Deployment
- Must address ONNX export, quantization, pruning, batching constraints, GPU memory consumption, and edge deployment factors.
